## Usage

Specifications are managed via wrangler MCP tools:
- `issues_create` with `type: "specification"` - Create new specifications
- `issues_list` with `types: ["specification"]` - List specifications
- `issues_search` - Search specifications

## Specification Workflow

1. **Draft** - Initial specification written
2. **Review** - Reviewed for completeness and feasibility
3. **Approved** - Ready for implementation planning
4. **Implemented** - Feature shipped and verified

## Creating Specifications

Use the `writing-specifications` skill to create well-structured specs that include:
- Problem statement
- Proposed solution
- Technical design
- Testing strategy
- Constitutional alignment
