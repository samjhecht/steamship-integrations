## DRY, KISS, YAGNI

**DRY (Don't Repeat Yourself):** Extract shared logic into a single authoritative location. But duplication is better than the wrong abstraction. Rule of Three: tolerate duplication twice; extract on the third occurrence.

**KISS (Keep It Simple):** Prefer straightforward solutions. If a junior developer cannot understand the code in under a minute, it is likely too complex.

**YAGNI (You Ain't Gonna Need It):** Implement only what is currently required. Do not build "just in case" abstractions, configuration layers, or feature flags unless a concrete need exists today.
