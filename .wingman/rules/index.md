## Content Placement Guide

| Content Type | Location | Loading | Budget |
|---|---|---|---|
| Universal rules (coding, testing, security) | `.wingman/rules/` | Via `.claude/rules/` symlinks (managed by `init_workspace --fix`) | <200 tokens each |
| Workspace conventions | `.wingman/config/conventions.md` | Always (via @import chain) | ~350 tokens |
| Path-scoped rules (domain-specific) | `.claude/rules/` with `paths:` frontmatter | Conditional (file-pattern match) | No hard limit |
| Reference material | `.wingman/memory/knowledge-base/` | Lazy (agent reads on demand) | No limit |
| Workflow instructions | Skills | On-demand (invoked) | No limit |

Rules are loaded via `.claude/rules/` symlinks managed by `init_workspace --fix`. Use `rules_list` and `rules_get` MCP tools for programmatic access.

To add a new rule: create a `.md` file in `.wingman/rules/` (or a subdirectory) and run `init_workspace --fix` to regenerate this index and sync symlinks.
